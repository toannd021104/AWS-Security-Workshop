#### Security Hub summary

1. Navigate to the **Summary** page in AWS Security Hub using the following link:  
   [AWS Security Hub summary page](https://console.aws.amazon.com/securityhub/home?#/summary)

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.2-Security-Hub-Dashboard/s1.png)

2. At the top of the page, review the **Security standards** widget.  
   This section shows your most recent overall security score as well as individual scores for each enabled Security Hub standard.  
   Security scores range from **0 to 100 percent** and represent the proportion of controls that have passed compared to all enabled controls.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.2-Security-Hub-Dashboard/s2.png)

3. Spend a few minutes reviewing the insights displayed on the Summary page.  
   These visualizations help identify which resources in your account are failing the most security checks.  
   One widget highlights the resources that have generated the highest number of findings, grouped by resource type.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.2-Security-Hub-Dashboard/s3.png)
![VPC](/images/2/2.1-AWS-Security-Hub/2.1.2-Security-Hub-Dashboard/s3b.png)

4. Scroll further down to the charts under **Most common threat type** and **Software vulnerabilities with exploits**.  
   These graphs summarize the dominant threat categories and known exploitable vulnerabilities detected across your environment.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.2-Security-Hub-Dashboard/s4.png)

To explore individual findings in more detail, review findings generated by **AWS GuardDuty**.  
Select one of the listed findings to open its detailed view, where you can examine the affected resource, severity, and contextual information.
