#### Configure SNS topic

1. Navigate to the Amazon SNS console.  
   https://console.aws.amazon.com/sns/v3/home

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s1.png)

2. From the left-hand navigation menu, select **Topics**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s2.png)

3. Click **Create topic**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s3.png)

4. Select the **Standard** topic type.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s4.png)

5. For the topic name, enter **security-hub-findings**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s5.png)

6. Leave all other settings at their default values and click **Create topic** at the bottom of the page.  
   This action creates the SNS topic that will receive Security Hub notifications.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s6.png)

#### Subscribe to the topic

7. From the **security-hub-findings** topic page, click **Create subscription**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s7.png)

8. On the **Create subscription** page, under **Protocol**, select **email**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s8.png)

9. Under **Endpoint**, enter the email address you want to use for this workshop to receive notifications.  
   You can unsubscribe after completing the workshop.

10. Click **Create subscription**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s10.png)

11. Check the inbox of the email address you provided.  
    Within a few minutes, you should receive a subscription confirmation email.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s11.png)

12. In the email, click **Confirm subscription** to activate the subscription.  
    This opens a confirmation webpage in your browser.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s12.png)

#### Create an EventBridge rule to send findings to the topic

13. After subscribing to the SNS topic, you are ready to route Security Hub findings to it.  
    Navigate to the Amazon EventBridge console.  
    https://console.aws.amazon.com/events/home

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s13.png)

14. Click **Create rule** on the right side of the page with **EventBridge Rule** selected.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s14.png)

15. On the **Define rule detail** page, name the rule **security-hub-findings**, then click **Next**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s15.png)

16. On the **Build event pattern** page, scroll to the **Event pattern** section and click **Edit pattern** in the bottom-right corner.

17. Add the following event pattern (JSON).  
    This pattern matches Security Hub findings with a **CRITICAL** severity level.

```js
{
  "source": [
    "aws.securityhub"
  ],
  "detail-type": [
    "Security Hub Findings - Imported"
  ],
  "detail": {
    "findings": {
      "ProductName": [
        "Security Hub"
      ],
      "Severity": {
        "Label": [
          "CRITICAL"
        ]
      }
    }
  }
}
```

18. Click **Next** to proceed to the target configuration step.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s18.png)

19. On the **Select target(s)** page, open the **Select a target** dropdown and choose **SNS topic** as the destination for the events.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s19.png)

20. From the **Topic** dropdown menu, select the previously created SNS topic named **security-hub-findings**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s20.png)

21. Click **Next** to continue.

22. On the **Configure tags â€“ optional** page, no tags are required for this workshop. Click **Next** to move forward.

23. On the **Review and create** page, review the rule configuration and click **Create rule**.  
    Monitor your email for the remainder of the workshop to observe notifications generated by Security Hub findings.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s23.png)

View the newly created rule:

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.5-Security-Hub-Notifications/s24.png)
