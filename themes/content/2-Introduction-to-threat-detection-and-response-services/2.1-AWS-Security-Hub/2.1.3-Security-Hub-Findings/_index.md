#### Findings in Security Hub

1. From the navigation panel on the left in Security Hub, select **Findings**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s1.png)

2. The Findings page supports filtering to help narrow down the list of displayed results.  
   Click **Add filter** to begin applying filters.

3. Select **Product name**, enter **GuardDuty** (case-sensitive), and then click **Apply**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s3.png)

4. The results now show all findings that Security Hub has ingested from the GuardDuty threat detection service.  
   At this stage, you should see multiple findings originating from GuardDuty.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s4.png)

To further refine the results, add an additional filter to focus on high-severity findings by clicking **Add filters** again.

5. From the dropdown menu, select **Severity label**, choose **is**, enter **HIGH** (case-sensitive), and click **Apply**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s5a.png)

Result:

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s5b.png)

6. Select one of the findings by clicking its title.  
   This opens the finding details panel. Expand all available sections and review the information, including the description, remediation guidance, affected resource details, and contextual metadata.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s6.png)

7. In the finding details panel, click the **Finding ID** at the top to display the complete JSON representation of the finding.  
   This JSON can be downloaded if required for deeper analysis or external investigation.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s7.png)

8. Close the JSON pop-up by clicking the **X** in the top-right corner.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s8.png)

9. In the finding details panel, open the **History** tab to view a chronological record of all updates made to the finding.  
   This history improves transparency and helps identify when and how the security issue evolved.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s9.png)

10. Close the finding details panel by clicking the **X** in the top-right corner, remaining on the **Findings** page.

11. To identify which resources are generating the highest number of findings, first remove all existing filters.  
    Then add a new filter: select **Resource type**, choose **is not**, enter **AwsAccount**, and click **Apply**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s11.png)

12. Add another filter by selecting **Record state**, choosing **is**, entering **ACTIVE**, and clicking **Apply**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s12.png)

13. Click **Add filters** again, then select **Group by** and choose **ResourceId**.  
    Click **Apply** to group findings by resource.  
    This view highlights which individual resources are responsible for the most security findings.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s13.png)

14. View detailed insight information by selecting **Insight details**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s14.png)
![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/s15.png)

15. Review additional findings filtered by the provider type **TTPs/Discovery/Recon:IAMUser-MaliciousIPCallerCustom**, generated by **GuardDuty**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/o1.png)

16. Select one of these findings to examine it in more detail.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/o2.png)

The finding indicates that an **ListBuckets** API call was made by unauthorized actors, identified through an IP address listed in a custom threat list.

17. Review additional findings generated by **Inspector**.

![VPC](/images/2/2.1-AWS-Security-Hub/2.1.3-Security-Hub-Findings/o3.png)

These findings are created when **Inspector** detects vulnerabilities during scans of EC2 instances.
